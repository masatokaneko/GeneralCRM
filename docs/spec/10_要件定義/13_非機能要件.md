# 13_非機能要件 (Non-Functional Requirements)

## 1. ユーザービリティ
*   **レスポンシブ対応**: デスクトップ、タブレット、モバイルの主要なブラウザ（Chrome, Safari, Edge, Firefox）で動作すること。
*   **操作性**: Salesforceに慣れたユーザーが直感的に操作できるナビゲーションとUIコンポーネントを提供する。

## 2. パフォーマンス (SLA)
*   **レスポンスタイム (目標値)**:
    *   API単件操作: 200ms 以内。
    *   一覧取得・リストビュー表示: 500ms 以内。
    *   複雑なレポート・集計: 同期実行時 2秒以内、それ以外は非同期処理。
*   **スケーラビリティ**:
    *   同時接続ユーザー数: Phase 1にて 1,000名の同時アクティブユーザーをサポート。
    *   データ処理性能: バルク処理にて 10,000件/分のスループットを維持。

## 3. セキュリティ
*   **認証 (Authentication)**:
    *   OAuth 2.0 / OpenID Connect 準拠。
    *   Bearer Token (JWT) によるステートレス認証。
    *   将来的な MFA (多要素認証) および IdP/SSO 連携のサポート。
*   **認可 (Authorization)**:
    *   X-Tenant-Id ヘッダーまたはJWT内包のテナントIDによる厳密なマルチテナント分離。
    *   OWD (組織内共有既定)、FLS (項目レベルセキュリティ)、ロール階層、共有ルールに基づくレコードレベルアクセス制御。
*   **通信・保護**:
    *   すべての通信は HTTPS (TLS 1.2以上) で暗号化。
    *   保存データの暗号化 (AES-256)、日次バックアップ。

## 4. 信頼性・可用性
*   **稼働時間**: 年間99.9%以上の稼働率を目指す。
*   **バックアップ**: 毎日1回のフルバックアップと、ポイントインタイムリカバリ (PITR) の可能なログ保存。

## 5. 拡張性・保守性
*   **APIファースト**: すべての機能はAPI経由で操作可能とし、外部システムとの統合を容易にする。
*   **ログ出力**: アプリケーションログ、アクセスログ、監査ログ（誰がいつ何を編集したか）の保持。

## 6. 法的・コンプライアンス
*   **GDPR/個人情報保護法**: 日本国内の個人情報保護法およびGDPRに準拠したデータ管理を可能にする。
