# 12_機能要件 (Functional Requirements)

## 1. 営業コア機能 (MVP: Must)

### 1.1 取引先・担当者管理
*   法人/個人の属性管理、Account階層表示。
*   Account/Contactの紐付けと、人物ベースの活動追跡。

### 1.2 リード管理
*   リードの登録、ステータス遷移（新規〜評価中）。
*   リード変換（取引先、連絡先、商談の原子的生成）。

### 1.3 商談管理
*   商談フェーズ(Stage)と確度(Probability)の連動。
*   パイプライン管理（カンバン表示、フェーズ別集計）。

### 1.4 商品・価格・明細 (Sales Core)
*   商品 (Product) および 価格表 (Pricebook) の管理。
*   商談品目 (OpportunityLineItem) による明細管理と金額自動計算。

## 2. 基盤・セキュリティ機能 (MVP: Must)

### 2.1 権限・共有モデル
*   OWD（組織内既定）による基本アクセス制御。
*   ロール階層による上位者へのデータ開示。
*   共有ルール（条件ベース・所有者ベース）による動的な権限拡張。
*   FLS（項目レベルセキュリティ）によるプロファイル別の表示・編集制御。

### 2.2 自動化 (Automation)
*   数式に基づくバリデーションルール（保存前エラーチェック）。
*   ワークフロールールによる項目自動更新、タスク作成、通知。

### 2.3 検索・クエリ
*   グローバル検索。
*   リストビューでのカスタムフィルタとビュー保存。

## 3. 分析・高度な自動化 (Phase 2: Should)

### 3.1 レポート・ダッシュボード
*   表形式・集計形式のレポート作成とグラフ化。
*   ダッシュボードによる主要KPIの可視化。

### 3.2 予測管理 (Forecasting)
*   売上予測カテゴリ別の積上げ集計。
*   マネージャーによる調整(Adjustment)機能。

### 3.3 見積・受注
*   見積 (Quote) の作成とOpportunity明細との同期。
*   商談または見積からの受注 (Order) 変換。

### 3.4 高度なプロセス
*   多段階の承認プロセス定義と実行。

## 4. インターフェース

### 4.1 REST API
*   完全なCRUD操作、バルクAPI、メタデータ取得。

### 4.2 GraphQL API (Phase 2)
*   柔軟なデータ取得を可能にするGraphQLエンドポイントの提供。
