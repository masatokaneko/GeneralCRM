# 23_画面レイアウト定義 (Screen Layout)

## 1. 共通レイアウト
すべてのメイン画面は以下の共通コンポーネントで構成されます。

*   **グローバルナビ（上部）**: ロゴ、グローバル検索バー、通知（ベル）、ユーザー設定。
*   **サイドバー（左側）**: ホーム、リード、取引先、連絡先、商談、ToDo、レポートへのリンク。

## 2. 一覧画面 (List View)
*   **ヘッダー**: オブジェクト名、選択中のビュー名（例：最近表示したデータ）、新規作成ボタン、インポートボタン。
*   **コントロールバー**: 検索（インクリメンタル）、フィルタ、表示形式切り替え（テーブル/カンバン）。
*   **データテーブル**:
    *   チェックボックス（一括処理用）。
    *   カラム（要件定義に基づいた主要項目。例：取引先名、都道府県、電話番号）。
    *   ソート機能（各カラム）。

## 3. 詳細画面 (Record Detail)
Salesforceの「Lightning Record Page」をモデルとした3ペイン構成。

### 3.1 左/中央ペイン（メイン情報）
*   **ハイライトパネル（上）**: 重要な5〜6項目、アクションボタン（編集、変換、削除等）。
*   **タブ領域**:
    *   **詳細**: 全項目の表示（セクション分け）。
    *   **関連**: 関連リスト（取引先なら連絡先、商談、活動など）。
    *   **ニュース/知見**: 外部ニュース等（将来拡張）。

### 3.2 右ペイン（活動・ユーティリティ）
*   **活動 (Activity)**: タスク作成、ログ記録、メール送信タブ。
*   **タイムライン**: 過去の活動履歴の時系列表示。
*   **チャット (Chatter相当)**: 社内SNS・コミュニケーション。

## 4. 編集/新規作成画面 (Edit/New Modal)
*   **構成**: モーダルウィンドウ形式。
*   **レイアウト**: 2列または1列のフォーム形式。
*   **機能**: 必須チェック、入力ヒント、即時バリデーション。
