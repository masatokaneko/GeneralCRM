# 11_ビジネス要件 (Business Requirements)

## 1. プロジェクト概要
Salesforce Sales Cloudのコア機能を踏襲しつつ、モダンで使いやすいCRM（顧客関係管理）システムを開発する。

## 2. ビジネス目標
*   顧客データの一元管理による営業効率の向上。
*   リードから商談成立までのプロセスの可視化と最適化。
*   データに基づいた売上予測の精度向上。

## 3. ターゲットユーザー
*   営業担当者 (Sales Representative)
*   営業マネージャー (Sales Manager)
*   システム管理者 (System Administrator)

## 4. 主なビジネス課題と解決策
| 課題 | 解決策 |
| :--- | :--- |
| リード情報の散逸 | リード管理機能による獲得情報の集中管理 |
| 商談の進捗が不透明 | 商談フェーズの定義と動的な管理画面 (パイプライン表示) |
| 顧客接点の履歴が残らない | 活動管理 (Task/Event) によるコミュニケーションログの記録 |

## 5. フェーズ計画 (Roadmap)

*   **第1フェーズ (MVP)**:
    *   リード、取引先、連絡先、商談の基本管理。
    *   商品 (Product)、価格表 (Pricebook)、商談品目 (OpportunityLineItem) のコア機能。
    *   セキュリティ基盤（OWD、ロール階層、共有ルール）。
*   **第2フェーズ**:
    *   売上予測 (Forecasting)、ダッシュボード、高度な自動化 (Approval/Workflow)。
    *   見積 (Quote) および同期機能。
    *   GraphQL API。
*   **第3フェーズ**:
    *   テリトリ管理 (Territory Management)。
    *   外部サービス連携 (Email, Slack等) の拡張。
    *   AIによる分析（Einstein相当の初期検討）。
*   **第4フェーズ**:
    *   高度なAI分析・予測。
    *   全社的なデータプラットフォーム化とエコシステム提供。
